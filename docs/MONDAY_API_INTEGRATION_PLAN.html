    Plan: Monday.com GraphQL API Integration for cori.data.rin

    Context

    The package currently requires a manually downloaded XLSX export from Monday board 6951894369 (group “Current”) placed in data/ before running the pipeline. The goal is to replace this manual step with a programmatic fetch via Monday’s GraphQL API, while keeping the XLSX as a fallback for cases where no API token is available.

    Only three columns from the Monday board are actually used downstream: name (item name), primary_county, other_counties.

    Files to modify

    • R/monday_api.R — new file: API fetch functions
    • R/load_data.R — modify load_rin_service_areas() to branch on API vs XLSX
    • DESCRIPTION — add httr2 to Imports
    • _targets.R — add "httr2" to tar_option_set(packages = ...)
    • params.yml — add comment noting MONDAY_API_TOKEN env var (no structural change)

    Step 1: Create R/monday_api.R

    Three functions:

    .check_monday_token() (private)

    Reads MONDAY_API_TOKEN env var; stops with a clear message if absent.

    .discover_monday_column_ids(board_id, token) (private)

    POST to https://api.monday.com/v2 with this query:

    query { boards(ids: [<board_id>]) { columns { id title } } }

    Returns a named character vector: c("Primary County" = "text0", "Other Counties" = "text1", ...). Stops if the board returns no columns or the response has errors.

    fetch_monday_board(board_id, group_title, col_primary_county, col_other_counties, page_limit) (exported)

    Phase A — call .check_monday_token() and .discover_monday_column_ids(). Resolve “Primary County” and “Other Counties” display names to internal column IDs. Stop with a descriptive error (listing available columns) if either is missing.

    Phase B — resolve group ID:

    query { boards(ids: [<board_id>]) { groups { id title } } }

    Filter groups by title == group_title; extract opaque id.

    Phase C — paginated item fetch. Initial page (cursor = NULL):

    query {
  boards(ids: [<board_id>]) {
    items_page(limit: <limit>, query_params: {rules: [{column_id: "group", compare_value: ["<group_id>"]}]}) {
      cursor
      items {
        name
        column_values(ids: ["<primary_id>", "<other_id>"]) { id text }
      }
    }
  }
}

    Subsequent pages use top-level next_items_page(limit: <limit>, cursor: "<cursor>") (not nested in boards). Loop until cursor is NULL or items is empty.

    Phase D — transform to data.frame:

    data.frame(name, primary_county, other_counties)

    text field from each column_values entry; NA when NULL or empty string.

    All httr2 calls use:

    httr2::request("https://api.monday.com/v2") |>
  httr2::req_method("POST") |>
  httr2::req_headers(
    "Authorization" = paste("Bearer", token),
    "API-Version"   = "2023-10",
    "Content-Type"  = "application/json"
  ) |>
  httr2::req_body_json(list(query = query_str)) |>
  httr2::req_perform()

    Always parse with httr2::resp_body_json(resp, simplifyVector = FALSE).

    Step 2: Modify load_rin_service_areas() in R/load_data.R

    Replace the readxl::read_excel(...) block (currently ~lines 150–156) with:

    monday_token <- Sys.getenv("MONDAY_API_TOKEN")

if (nchar(monday_token) > 0) {
  message("MONDAY_API_TOKEN found — fetching board data via API")
  rin_only <- tryCatch(
    fetch_monday_board(
      board_id    = 6951894369,
      group_title = params$monday_network_communities_tab_name
    ),
    error = function(e) {
      warning("Monday API fetch failed: ", conditionMessage(e),
              "\nFalling back to XLSX file.", call. = FALSE)
      NULL
    }
  )
  if (is.null(rin_only)) monday_token <- ""
}

if (nchar(monday_token) == 0) {
  message("No MONDAY_API_TOKEN — reading XLSX: ", params$monday_network_communities_file_name)
  rin_raw <- readxl::read_excel(
    paste0(data_dir, "/", params$monday_network_communities_file_name), skip = 2
  )
  names(rin_raw) <- snakecase::to_snake_case(names(rin_raw))
  rin_only <- rin_raw |>
    dplyr::filter(!is.na(`name`)) |>
    dplyr::filter(`name` != "Subitems")
}

    The API path already produces columns named name, primary_county, other_counties. All downstream code in load_rin_service_areas() is unchanged.

    Set MONDAY_API_TOKEN in ~/.Renviron (or the project’s _environment.local, which is already gitignored):

    MONDAY_API_TOKEN="<token_from_monday_admin>"

    Verification

    1. API path: Set token, rename XLSX, run tar_make(rin_service_areas). Compare row count and community names to previous output.
    2. XLSX fallback: Unset token, restore XLSX, run tar_make(rin_service_areas). Output must be identical to current production data.
    3. API failure fallback: Set token to invalid value, restore XLSX. Confirm warning is emitted and pipeline completes via XLSX.
    4. Missing column error: Call fetch_monday_board(col_primary_county = "Bad Name") manually. Confirm stop message lists available column names.
